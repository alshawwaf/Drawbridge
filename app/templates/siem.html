On the Management Server, add an export target. The receiver auto-detects all of these — pick whichever your SIEM expects and the command updates. The export name carries the format + protocol, so you can point several exporters at the portal at once without name collisions:

FORMAT {% for f in ['cef', 'leef', 'json', 'syslog', 'generic', 'splunk', 'logrhythm'] %}{% endfor %} PROTOCOL {% for p in ['udp', 'tcp'] %}{% endfor %} 
cp_log_export add name dcsim-siem-cef-udp \
  target-server {{ host }} target-port {{ port }} \
  protocol udp format cef
cp_log_export restart name dcsim-siem-cef-udp

Listening on {{ host }}:{{ port }} (TCP + UDP). On a hosted instance, make sure that port is reachable from the Management Server — the bundled docker-compose.yml publishes it; on Dokploy add a TCP+UDP entrypoint (see deployment notes). No gateway yet? Click Send test log.
