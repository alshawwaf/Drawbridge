MCP for agents

Drive Drawbridge from an LLM agent (n8n, Claude Desktop, Cursor, VS Code) over the Model Context Protocol. The agent gets typed tools to reason about and act on Check Point access policy — decide on a request, correlate a service/app name, analyse a layer, and (gated) apply a change. Same idea as Check Point's MCP servers, focused on the access-automation engine.

Safety

  
Every call needs Authorization: Bearer <token>. The token is set in
  Settings → MCP / agent (stored encrypted at rest; a DCSIM_MCP_TOKEN env var is used
  only as a fallback) and can be set, rotated, or cleared with no redeploy — no token configured →
  503 (endpoint disabled), wrong/missing token → 401. decide_access, the analyse tools,
  and the correlate/coverage tools are read-only. apply_access with publish=false
  dry-runs (validate then discard); publish=true is refused unless an admin enables
  Settings → MCP / agent → "Let the MCP agent publish to live policy". The engine's own guarantees
  still hold — an unknown/ambiguous service returns a review + suggestions, never a wrong call to the SMS.
