Ticketing webhook (any ITSM — end-to-end automation)

Point any system that can POST JSON at the endpoint below — ServiceNow, Jira, Remedy, Freshservice, or a custom portal. Set DCSIM_WEBHOOK_TOKEN and send it as X-DCSim-Token; the webhook is disabled until that token is set. Field names are flexible (common u_* / number / Jira key aliases are accepted). With "apply": true it applies + publishes. The result is returned synchronously; to push it back, add an optional "callback_url" (we POST the result there) or configure the built-in ServiceNow write-back (DCSIM_SERVICENOW_*).

How it decides — behind the scenes

Every request runs the same first-match decision tree the market leaders use. It reuses a rule that already permits the traffic; widens a rule only when it matches the request exactly in the other two columns (so it never grants access nobody asked for); creates a least-privilege rule when nothing fits; and refuses to touch policy it can’t safely reason about.
