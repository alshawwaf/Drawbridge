About “Try it out”. By default it runs through the portal (server-side), so it reaches your saved Management/Gateway servers without the browser's cross-origin (CORS) block — just Authorize with a session id (sid) and execute. The proxy only ever reaches servers you've added, and TLS is verified server-side. Untick “Run calls through the portal” to call straight from the browser instead — handy for the copy-paste curl, though a real server will then show “Failed to fetch — CORS” (expected, not a portal bug). For governed changes, use the gated editor on Management Servers (preview → publish). You can also Export for Postman / Bruno.

Export for Postman / Bruno downloads this version's OpenAPI 3.0 file with your selected target server baked in. In Postman: Import → File (builds a collection of every command). In Bruno: Import Collection → OpenAPI V3. Re-export after switching the version or target server to get a fresh file.
