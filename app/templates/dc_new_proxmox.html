Proxmox API token

If set, the mock validates the Authorization: PVEAPIToken=<id>=<secret> header and returns 401 on mismatch. Stored only as a one-way hash. In Proxmox the token's user needs Sys.Audit, VM.Audit, VM.Monitor (e.g. the PVEAuditor role).

In SmartConsole, the Token ID must be the real user@realm!name format (e.g. root@pam!cloudguard) even if you leave the secret blank here — CloudGuard parses it, and a bare value like any fails Test Connection with "Data Center service problem".
